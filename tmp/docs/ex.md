# TTL 인덱스를 활용한 경로 데이터 자동 만료 시스템
## Situation
- 대중교통 경로 검색 결과를 DB에 저장한 뒤, 이후 상세 안내 시 재활용하는 구조 
- 24시간 주기 Cron Job 기반 일괄 삭제 방식으로 인해 특정 시간대에 삭제 쿼리가 집중되며 DB 부하 스파이크가 발생 
- 삭제 스크립트 장애 또는 누락 시 데이터가 지속적으로 남을 수 있는 구조적 리스크 존재
## Action
- createdAt 필드에 TTL 인덱스를 적용해 일정 시간이 경과한 데이터가 자동으로 만료되도록 설계
- MongoDB TTL 모니터의 백그라운드 삭제 메커니즘을 활용해 일괄 삭제 구조를 제거하고, 삭제 부하를 지속적으로 분산
- Cron Job 기반 삭제 로직을 제거해 배치 스크립트 장애에 대한 운영 의존성 해소
## Result
- 특정 시간대에 집중되던 대량 삭제 작업이 제거되어 DB 부하 스파이크 해소
- 데이터 수명 관리가 애플리케이션 로직이 아닌 DB 레벨에서 자동으로 처리되는 구조 확보

# MongoDB 2dsphere 인덱스를 활용한 위치 기반 검색 최적화
## Situation
사용자 현재 위치 기준 반경 N km 내 엘리베이터·리프트를 검색하는 기능 필요
공공 데이터 + 더미 데이터(약 1,000,000건)를 애플리케이션으로 불러온 뒤 메모리에서 필터링하는 방식으로, 평균 응답 시간이 약 2~3초 수준
## Action
MongoDB explain()을 활용한 쿼리 플랜 분석을 통해 COLLSCAN 발생 및 인덱스 미사용 병목 원인 식별
위치 데이터 스키마를 GeoJSON Point 표준({ type, coordinates })으로 마이그레이션
위치 필드에 2dsphere 인덱스를 적용하여 구면 기하 기반 지리 검색 지원
$geoWithin + $centerSphere 조합을 활용한 원형 반경 검색 쿼리 설계 및 구현
## Result
100만 데이터 기준, 쿼리 실행시간 약 150배 (2,000~3,000ms → 10~20ms) 단축
